# Dataset Inventory

## 목적
이 문서는 캡스톤 프로젝트에서 사용하는 원천 데이터셋의 구성, 출처, 용도, 전처리 우선순위를 정리하기 위한 문서이다.

프로젝트의 핵심은 실시간 멘토링 환경에서 채팅 중 질문을 추출하고, 중복을 제거한 뒤, 멘토가 답변하기 좋은 질문을 우선 정렬하는 시스템을 구현하는 것이다. 따라서 데이터셋도 질문 탐지, 채팅체 이해, 멀티턴 대화 맥락 분석에 도움이 되는지를 기준으로 관리한다.

## 2. 데이터셋별 상세 메모

### 2.1 kor_3i4k
- **경로**: `data/raw/3i4k/kor_3i4k/`
- **현재 확인 구조**
  - `data/`
  - `train-00000-of-000..`
  - `test-00000-of-000..`
  - `.gitattributes`
  - `README.md`
- **예상 특징**
  - Hugging Face 스타일로 저장된 데이터셋일 가능성이 높음
  - train/test 분할이 이미 존재
- **활용 가설**
  - 질문문/대화문 표현 패턴 보조 학습
  - 한국어 비격식 문장 처리 보조
- **추가 확인 필요**
  - 실제 컬럼 구조
  - 질문 관련 라벨 존재 여부
  - 라이선스 및 사용 조건

### 2.2 AIHub Online Spoken
- **경로**: `data/raw/aihub_online_spoken.../`
- **현재 확인 구조**
  - `라벨링데이터/`
  - `원천데이터/`
  - 압축파일 포함
- **예상 특징**
  - 온라인/구어체 발화 중심 데이터
  - 발화 단위 분석에 적합할 가능성 있음
- **활용 가설**
  - 실시간 채팅과 완전히 같지는 않지만, 구어체 질문 패턴 학습에 도움
  - 질문 탐지 분류기용 보조 데이터셋으로 활용 가능
- **추가 확인 필요**
  - 라벨 정의
  - json/csv 구조
  - 질문/비질문으로 직접 변환 가능한지 여부

### 2.3 AIHub SNS Multiturn
- **경로**: `data/raw/aihub_sns_multiturn.../`
- **현재 확인 구조**
  - `Other/`
  - `SubLabel/`
  - `Training/01.원천데이터`
  - `Training/02.라벨링데이터`
  - `Validation/`
- **예상 특징**
  - SNS 스타일의 멀티턴 대화
  - 채팅체, 짧은 응답, 대화 문맥 반영 가능성 높음
- **활용 가설**
  - 질문 탐지 데이터셋의 핵심 후보
  - 멀티턴 맥락 기반 질문 처리 실험 가능
  - 향후 질문 추천/정렬에서 문맥 feature 설계에 참고 가능
- **추가 확인 필요**
  - 질문 관련 직접 라벨 여부
  - 대화 turn 구조
  - speaker 정보 포함 여부
  - 학습/검증 분할 형식

## 3. 전처리 우선순위

### 1순위: AIHub SNS Multiturn
이유:
- 실제 서비스 입력이 실시간 채팅에 가까움
- 멀티턴 구조를 포함해 질문이 문맥 속에서 등장하는 상황을 반영하기 좋음

### 2순위: AIHub Online Spoken
이유:
- 구어체 발화 데이터로 질문 표현 다양성을 보완할 수 있음
- STT 이후 발화문 처리에도 간접적으로 참고 가능

### 3순위: kor_3i4k
이유:
- 보조적 한국어 대화 패턴 데이터로 활용 가능
- 질문 탐지 전용 데이터인지 추가 확인이 필요함

---

## 4. 활용 계획

### 질문 탐지
- 입력 문장이 질문인지 아닌지 분류
- 규칙 기반 필터 + 경량 분류기 학습 데이터 후보로 사용

### 중복 제거
- 의미적으로 유사한 질문 표현 수집
- 임베딩 기반 유사도 threshold 실험용 예시 확보

### 질문 정렬
- 멀티턴 문맥 데이터에서 직전 발화와 질문의 관련성을 살펴보는 참고 자료로 활용

---

## 5. 관리 원칙
1. 원본 데이터는 `data/raw/`에서 직접 수정하지 않는다.
2. 압축 해제 및 정규화 결과는 `data/interim/`에 저장한다.
3. 학습용 최종 산출물은 `data/processed/`에 저장한다.
4. 데이터셋별 라이선스와 출처는 추후 별도로 보강한다.
5. 실제 모델 학습에 사용한 파일은 재현 가능하도록 경로와 생성 스크립트를 함께 기록한다.
